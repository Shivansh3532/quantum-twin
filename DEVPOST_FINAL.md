## Tagline

Competing post-quantum migrations, selected by external deterministic evidence.

## Short description

Quantum Twin gives the same immutable repository contract to two isolated Codex builders—Direct Cutover and Compatibility Bridge—then independently evaluates both twice and deterministically selects only an eligible migration.

## Full project description

Quantum Twin turns post-quantum migration from one generated patch into a constraint-driven tournament whose winner is selected by external deterministic evidence.

It recursively discovers native RSA signing and verification in a trusted local Node repository, freezes a versioned contract, and starts two real `@openai/codex-sdk@0.144.6` builders with exact `gpt-5.6-sol`. Direct Cutover targets ML-DSA-65 only. Compatibility Bridge targets ML-DSA-65 while retaining RSA output required by a declared frozen consumer.

A copied external compatibility harness runs twice. Deterministic TypeScript verifies commands, original tests, protected and writable boundaries, dependencies, native crypto use, ML-DSA verification, tamper rejection, wrong-key rejection, domain context, legacy compatibility, repeatability, diffs, and hashes. Failed candidates are ineligible. If none qualify, Quantum Twin returns NO SAFE WINNER.

The public read-only explorer compares two genuine CommonJS/npm release-fixture runs. Compatibility required selects Bridge after Direct fails compatibility twice. Compatibility disabled selects Direct because both pass and Direct retains fewer RSA signatures.

## Problem

Discovery tools identify legacy cryptography, while a single coding-agent prompt can create one plausible patch. Neither alone proves that a concrete migration preserves declared compatibility or passes negative checks.

## Solution

Generate competing implementations from one frozen contract, isolate builders, evaluate both with code outside their worktrees, and select only from verified eligible candidates.

## How it works

1. Recursive scanner records supported RSA signing and verification evidence.
2. Versioned contract freezes source, commands, writable/protected paths, ML-DSA-65 context, compatibility requirement, external harness, dependency policy, and limits.
3. Two isolated Codex SDK threads implement Direct and Bridge strategies.
4. External deterministic gates run twice and hash evidence.
5. TypeScript selects an eligible candidate or returns NO SAFE WINNER.
6. GPT-5.6 explains immutable results after selection.

## Exact Codex use

`@openai/codex-sdk@0.144.6` starts two isolated, no-network, approval-never worktree builders with the same repository contract. Candidate thread IDs, commits, diffs, and durations are public in genuine reports. Codex also accelerated scanner generalization, safe repository intake, UI implementation, testing, and documentation during Build Week.

## Exact GPT-5.6 use

Exact `gpt-5.6-sol` produces schema-validated RSA classification and post-selection explanation. Deterministic post-validation requires GPT operations and affected files to equal scanner evidence; contradiction fails classification. GPT never controls gates, measurements, eligibility, or selection. Direct Responses API was not used because `OPENAI_API_KEY` was unavailable.

## Technological implementation

Node.js 24.18.0, TypeScript, Next.js, Zod, native `node:crypto` ML-DSA-65, Git isolated copies/worktrees, copied external harnesses, two evaluator passes, report/diff/config/manifest/harness hashes, Vitest negative cases, deterministic report CLI, and Windows/Ubuntu GitHub Actions.

## Design

Responsive evidence workspace with immediate two-scenario comparison, genuine-run identity, scanner findings, candidate status, full diffs, command evidence, measurements, gate matrix, provenance, hashes, GPT limitations, scope boundaries, keyboard-visible focus, native selector, aria-live changes, and read-only downloads.

## Potential impact

Application security engineers and maintainers of Node services with deployed RSA verifiers can compare staged migration implementations against explicit compatibility instead of accepting one opaque rewrite. NIST standardized the first post-quantum cryptography algorithms in 2024 and urges migration; NCCoE separates discovery and interoperability testing; CISA/NSA/NIST urge early planning and cryptographic inventory.

## Novelty

Traditional scanners locate cryptography. One coding-agent prompt yields one candidate. Quantum Twin creates isolated competing implementations from the same contract, independently disqualifies failures, changes the selected result when declared compatibility changes, preserves complete provenance, and supports NO SAFE WINNER.
