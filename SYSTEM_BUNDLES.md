# System Bundles

A System Bundle is a generated run artifact, not a prerequisite file. `/lab` accepts a name, 1–12 public GitHub or trusted local repositories, and named frozen consumers. Each repository is copied into private ignored storage and recorded with sanitized source identity, exact commit when available, normalized tree SHA-256, package manager, module type, entry points, commands, components, ports, health checks, and dependencies.

Deterministic discovery reads package scripts, npm/pnpm/Yarn lockfiles, workspace declarations, source entry points, tests, and Compose files. The generated contract records `observed-source`, `observed-runtime`, `inferred-deterministically`, `suggested-gpt`, or `supplied-user` provenance. GPT suggestions never approve a field. The user reviews the graph and exact hash, types `APPROVE SYSTEM CONTRACT`, then separately grants repository/command/Codex authority and types `RUN COORDINATED TOURNAMENT`.

All repositories are assembled under `repositories/<id>` in one isolated Git baseline so a builder can coordinate producers and consumers transactionally. Reports and patches remain grouped by repository. Original source paths are never written. A stale tree or changed contract hash invalidates approval.

The evidence download includes repository-grouped patches and the rollout package. Optional local application is a separate CLI boundary: the user supplies an explicit repository-ID-to-clean-Git-root map and types `CREATE COORDINATED LOCAL BRANCHES`. Quantum Twin preflights every patch, creates `quantum-twin/*` branches through temporary Git worktrees, commits only there, never switches or writes `main`, and never pushes.

Current limitations are generated by [`support-matrix.json`](support-matrix.json). Docker Compose relationships are discovered, but Docker execution remains experimental until complete cross-platform proof exists.
